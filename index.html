How the engine works

Weighted graph. Every road is an edge whose cost = distance × live traffic multiplier (1.0–3.0). Cities are nodes.

Dijkstra's algorithm expands the lowest-cost unvisited node first, guaranteeing the optimal path.

Fastest vs Shortest. Fastest minimises traffic-weighted cost. Shortest minimises raw km — it may be slower but uses less fuel.

Alternate routes use a simplified Yen's k-shortest-paths algorithm — it blocks edges from found paths segment by segment and re-runs Dijkstra.

ETA is always computed from actual traffic-weighted time so you can compare across modes.
